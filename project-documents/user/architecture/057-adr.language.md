# ADR 057: Language — Python 3.11+

## Context

Office Hero requires a backend language for the REST API, business logic,
data access, and MCP server. The primary developer is Jake, a solo developer
working with AI co-pilots.

## Decision

**Python 3.11+**

## Rationale

| Factor | Python 3.11+ | Go | Node.js (TypeScript) |
| ------ | ------------ | -- | -------------------- |
| FastAPI ecosystem | Native | N/A | N/A |
| SQLAlchemy async ORM | Native | N/A | N/A |
| Type hints | Native (PEP 484+) | Native | Via TypeScript |
| AI/ML libs (future) | Unmatched | Limited | Limited |
| Solo dev productivity | High | Medium | Medium |
| Pre-commit / quality | Mature (black, ruff, bandit) | Good | Good |
| 3.11 performance | 60% faster than 3.10 | Faster | Faster |
| MCP SDK support | Official Python SDK | Community | Official SDK exists |

Python 3.11 specifically adds:

- `tomllib` in stdlib (used by pyproject.toml tooling)
- Task groups in asyncio (`asyncio.TaskGroup`)
- Fine-grained error locations in tracebacks (developer productivity)
- 60% average performance improvement over 3.10

## Isolation Strategy

**The language choice is architectural — but the seams are already isolated:**

- Business logic lives in `services/` — no HTTP, no DB imports. If business logic
  needed to move to another language (e.g., performance-critical path to Go), this
  layer has clear boundaries.
- API contracts are defined by the OpenAPI spec — any language can implement it.
- Data contracts are defined by Pydantic schemas and the DB schema — language agnostic.
- The MCP server is a separate process — can be rewritten in another language
  without touching the API.

**To replace Python with another language:**
Replace `src/`, `pyproject.toml`, `Dockerfile` — the OpenAPI spec, DB schema,
and `.env`-based config remain unchanged. Frontend and mobile are unaffected.

## Consequences

- `pyproject.toml` is the single project config file (deps, tool settings, test config)
- All tooling (black, ruff, bandit, pytest, pip-audit) assumes Python
- `pydantic-settings` manages all config from env vars
- GitHub Actions CI targets Python 3.11
- Dockerfile uses `python:3.11-slim`
