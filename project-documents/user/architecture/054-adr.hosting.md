# ADR 054: Hosting — Fly.io (app) + Neon (database)

## Context

Budget constraint: free hosting as long as possible. Revisit when profitable.
The app needs: always-on API, background location processing, static frontend assets.
The database needs: PostgreSQL 15+, RLS support, reliable free tier, dev branching.

## App Hosting Decision: Fly.io

| Factor | Fly.io | Render | Railway |
| ------ | ------ | ------ | ------- |
| Free tier always-on | Yes (3 shared VMs) | No (spins down after 15 min) | Deprecated free tier |
| Cold starts | None | Yes (~30 s) | N/A |
| Background workers | Yes | Paid | Paid |
| Docker support | Yes | Yes | Yes |
| Python support | Excellent | Good | Good |
| Scale-up path | Easy | Easy | Easy |

Render's cold starts would cause 30-second delays on the first request after idle —
unacceptable for a dispatch tool. Fly.io runs continuously on the free tier.

**Decision: Fly.io.** Deploy as a Docker container. App and MCP server run as separate
Fly.io apps sharing the same Neon database.

## Database Hosting Decision: Neon

| Factor | Neon | Supabase | Fly.io Postgres |
| ------ | ---- | -------- | --------------- |
| Free tier pause | Never | After 7 days inactivity | N/A |
| Branching | Yes (great for dev/test) | No | No |
| Serverless | Yes (auto-scales to 0) | No | No |
| RLS tooling | CLI / migrations | GUI + CLI | CLI |
| Free storage | 512 MB | 500 MB | 1 GB |
| Managed | Yes | Yes | No (self-manage) |

Neon's branching means each developer (or CI run) can have its own database branch —
critical for isolated integration testing without affecting production.
Alembic manages all RLS policies via migrations, so no GUI tooling is required.

**Decision: Neon.** Primary branch for production; `dev` branch for local development;
ephemeral branches for CI integration tests.

## Consequences

- Fly.io free tier: 3 × 256 MB RAM shared VMs, 3 GB storage, 160 GB bandwidth/month
- Neon free tier: 512 MB storage, 10 branches, auto-suspends compute after 5 min
  idle (instantly resumes on connection — no cold start visible to users)
- Static frontend assets: served from Fly.io or Cloudflare Pages (free)
- When revenue justifies: upgrade Fly.io to dedicated VMs; upgrade Neon to paid plan
