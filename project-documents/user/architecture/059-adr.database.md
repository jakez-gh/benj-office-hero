# ADR 059: Database — PostgreSQL 15+

## Context

Office Hero requires a relational database that supports:

- Multi-tenant row-level isolation (hard security requirement)
- JSONB for industry-specific `custom_fields`
- Time-series data (vehicle location history)
- Async queries (non-blocking I/O)
- Free/low-cost hosting at launch

## Decision

**PostgreSQL 15+ hosted on Neon (serverless, free tier).**

## Rationale

| Factor | PostgreSQL 15+ | MySQL 8 | SQLite |
| ------ | -------------- | ------- | ------ |
| Row-Level Security (RLS) | Native | No | No |
| JSONB column type | Native | JSON (limited) | No |
| Async driver | asyncpg | aiomysql | aiosqlite |
| Full-text search | Native | Limited | Limited |
| Window functions | Full | Limited | Basic |
| Neon hosting | Yes (free tier + branching) | No | No |
| Transactional DDL | Yes | No | Yes |

**PostgreSQL RLS is non-negotiable** for our tenant isolation model (see ADR 053).
No other open-source database supports row-level security policies with
session-variable tenant context.

**JSONB** enables the `custom_fields` column to store plumbing, HVAC, and pest
control specific fields without separate tables per industry — critical for
the multi-industry data model.

## Isolation Strategy

**PostgreSQL is isolated behind the Repository layer and SQLAlchemy dialect:**

- Repositories are the only code that issues queries
- SQLAlchemy dialect handles PostgreSQL-specific SQL generation
- Pydantic schemas define the data contracts — independent of the DB engine
- The OpenAPI spec and REST API are completely DB-agnostic

**DB-specific features and their swap cost:**

| Feature | Swap cost | Mitigation |
| ------- | --------- | ---------- |
| RLS policies | High | Alembic manages; service-layer `tenant_id` check is fallback |
| JSONB columns | Medium | Could serialize to TEXT JSON in another DB |
| `gen_random_uuid()` | Low | Application-side UUID generation as fallback |
| `TIMESTAMPTZ` | Low | Standard SQL type |

**To replace PostgreSQL:**

1. Implement tenant isolation at service layer only (remove RLS dependency)
2. Replace JSONB with serialized JSON in TEXT
3. Change SQLAlchemy dialect
4. Rewrite Alembic migrations

This is a significant change but confined to `repositories/`, `models/`, and `db/`.

## Consequences

- `DATABASE_URL` must use `postgresql+asyncpg://` scheme
- RLS requires `SET LOCAL app.tenant_id = '...'` before every query (see ADR 053)
- Neon free tier: 512 MB storage, auto-suspends compute (not the app — see ADR 054)
- `outbox_events` and `saga_log` tables use PostgreSQL-specific `gen_random_uuid()`
- `TIMESTAMPTZ` used for all timestamps (timezone-aware)
- Neon branching used per CI run for integration tests (branch per PR/run, deleted after)
