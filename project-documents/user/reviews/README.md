# user/reviews/ — Directory Inventory

Holds code review documents and associated follow-up items. These are often
generated by Claude Code agents or human reviewers during development.

## What goes here

- Review reports (900-939 range) describing findings and actions
- Tasks or todos created as a result of reviews

## What does NOT go here

- Task breakdowns (go in `user/tasks/`)
- Slice designs or architecture docs

Keep review documents concise; once issues are resolved they can be archived
elsewhere or deleted.
